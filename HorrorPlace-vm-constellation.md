### Registry-backed resolution

Under the hood, each API call resolves IDs against the public registries in `Horror.Place`, which in turn point to signed underground descriptors. For example, `H.invariants(region_id, tile_id)` is specified to load a normalized invariant bundle via `registry/regions.json` and `registry/invariants.json`, with hashes and provenance enforced by the orchestrator workflows.
```lua
-- horror_api_impl.lua (engine-side illustration, not public-contract)

local Registries = require("horror_registries")  -- wraps JSON registry access
local Bundles    = require("horror_bundles")     -- wraps invariant bundle loading/verification

function H.invariants(region_id, tile_id)
    local region_entry = Registries.region(region_id)
    local bundle_ref   = region_entry and region_entry.invariants
    return Bundles.load_tile(bundle_ref, tile_id)  -- verifies hash/signature before returning
end

function Director.query_events(args)
    local inv = H.invariants(args.region_id, args.tile_id)
    local metrics = H.session_metrics(args.session_id)
    -- Filter events based on CIC/RRM/SPR/AOS and entertainment targets UEC/EMD/STCI/CDL/ARR.
    return Bundles.events_for(inv, metrics, args)
end
```

New horror content enters the system only by adding or updating contract files and registry entries via the automated, cryptographically verified workflows described earlier. ## Declarative wiring

Cross-repository linkage is expressed declaratively through schemas, registries, and DSLs instead of imperative glue code. Every object in the constellation—events, regions, personas, styles—is defined by a JSON schema in the public core, such as `schemas/invariants_v1.json`, `schemas/entertainment_metrics_v1.json`, and `schemas/stylecontract_v1.json`. Registries like `registry/events.json`, `registry/regions.json`, `registry/styles.json`, and `registry/personas.json` map stable IDs to contract locations and metadata rather than raw file paths. Runtime systems and AI agents resolve objects by ID through these registries, which allows asset locations to change without breaking consumers.
