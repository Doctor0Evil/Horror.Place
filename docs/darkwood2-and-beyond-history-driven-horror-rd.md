## What Horror.Place actually provides

Horror.Place is **not** a content pack and **not** a monolithic engine. It is a small, sharp set of pieces that can sit beside any existing codebase:

- **Schemas and contracts, not scenes.**  
  JSON Schemas for safety invariants (CIC, AOS, RRM, SHCI, etc.), entertainment metrics (UEC, EMD, STCI, CDL, ARR), style contracts, and event/persona structures.  These define *what* a scare is allowed to draw from: history, archival gaps, ritual residue, liminal stress—never raw gore.
